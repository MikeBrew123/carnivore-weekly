THE ISSUE IN PLAIN ENGLISH
================================================================================

The banners on wiki.html and calculator.html are PERFECTLY CONSISTENT with
each other (they use the same CSS file). However, they both have the SAME
styling errors because they share the CSS.

The problems are in: /public/style-2026.css

WHAT'S WRONG:

1. The "CARNIVORE WEEKLY" heading appears in tan (#d4a574) instead of the
   bright gold (#ffd700) that the brand requires. This makes the heading
   appear less prominent and violates brand standards.

2. The "CARNIVORE WEEKLY" heading doesn't explicitly use the Playfair Display
   font, even though it's imported. As a result, it falls back to the browser's
   default serif font instead of the brand serif font.

NOTE: The subtitle "The Meat-Eater's Digest" IS correctly colored gold,
which makes the H1 inconsistency even more obvious.

================================================================================
THE FIX (2 Changes, 1 File)
================================================================================

FILE: /Users/mbrew/Developer/carnivore-weekly/public/style-2026.css

LOCATION: Around line 127 (header h1 rule)

CURRENT CODE (WRONG):
```
header h1 {
    color: #d4a574;
    font-size: 4em;
    margin: 15px 0 10px 0;
    font-weight: 900;
    letter-spacing: 4px;
    text-shadow: 3px 3px 6px rgba(0,0,0,0.7);
}
```

CORRECTED CODE:
```
header h1 {
    color: #ffd700;  /* CHANGE: from #d4a574 to #ffd700 */
    font-family: 'Playfair Display', serif;  /* ADD: this missing property */
    font-size: 4em;
    margin: 15px 0 10px 0;
    font-weight: 900;
    letter-spacing: 4px;
    text-shadow: 3px 3px 6px rgba(0,0,0,0.7);
}
```

IMPACT:
  - Fixes H1 color on BOTH wiki.html and calculator.html
  - Fixes H1 font-family on BOTH wiki.html and calculator.html
  - One file edit fixes both pages
  - Improves accessibility (gold provides better contrast on brown)
