## COMMON QUESTIONS

**Q: Which document should I read first?**
A: `/docs/sarah-positioning-executive-summary.md` Always. It's the gateway to everything.

**Q: Can I just use the implementation guide without reading the voice guide?**
A: Not recommended. You'll build the structure right but write/position it wrong. Read Section 1 first.

**Q: Where's the CSS for Sarah's section?**
A: `/docs/sarah-positioning-implementation.md` → Parts 1 and 3

**Q: How do I know if I'm doing this right?**
A: Use the relevant checklist in `/docs/sarah-positioning-executive-summary.md`

**Q: What if I can't decide between Option A and B visual treatment?**
A: Read Section 2.2 in `/docs/sarah-voice-positioning.md` for pros/cons of each.

**Q: How often should I reference these documents?**
A: Writers/designers: Before every piece. Developers: During implementation. Everyone: Monthly for brand audits.

**Q: Can I create my own version of Sarah's voice?**
A: No. These documents define the voice to ensure consistency. Personalization happens within these guardrails.

**Q: What if I find an error or unclear section?**
A: Document it, escalate to CEO. These are living documents that can be improved.
