Decision Rules (Use These)

‚úÖ Use 1 Agent When:

‚Üí Task is isolated to specific files/area
‚Üí Question can be answered in one pass
‚Üí No dependencies between searches
‚Üí Simple verification or exploration
Example: "Find all instances of .logo class"

‚úÖ Use 2-3 Agents When:

‚Üí Multiple independent areas to explore
‚Üí Task has 2-3 distinct components
‚Üí No dependency between findings
‚Üí All agents can run in parallel
Example: Verify HTML validity + logo size + nav duplication (3 agents)

‚úÖ Use 4+ Agents (Parallel) When:

‚Üí Running full validation suite
‚Üí Each validator is independent
‚Üí All can execute simultaneously
‚Üí Waiting for all results before proceeding
Example: copy-editor + brand + humanization + structural + visual

‚ùå DON'T Use Agents When:

‚Üí Simple single-line edits
‚Üí Trivial file reads
‚Üí User explicitly asked to see command output
‚Üí Debugging (user needs details)
Example: Quick grep, simple read, one-line fix

Your Workflow (When You See a Task)

1
Understand the problem - What needs to be done? (OK to think/explore)
2
Design the solution - What's the approach? (OK to use Plan agent)
3
Decide: Who does the work? - You or an agent? (Use decision matrix above)
4
Delegate & coordinate - Launch agents in parallel if possible
5
Review results - Did agents find what we needed?
6
Make decisions - Based on findings, what's next?
